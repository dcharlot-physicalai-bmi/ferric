# ferric-ffi — the Ferric C ABI (`libferric`)

The universal on-ramp. Any language with C FFI drives the pure-Rust cross-fabric runtime:

```c
FerricHandle* h = ferric_load("model.gguf");
char* text = ferric_generate(h, "The capital of France is", 8);          // free text
char* json = ferric_generate_json(h, "A person.", "{...json-schema...}", 40); // guaranteed-conformant
ferric_free_string(text); ferric_free_string(json); ferric_free(h);
```

Build `libferric.{dylib,so,a}` with `cargo build -p ferric-ffi --release`; the header
`libferric.h` is generated by `cbindgen`. Bindings (see `bindings/`):

- **Zig** — `@cImport("libferric.h")`, zero binding code (verified).
- **Mojo** — Python interop (the `ferric` package) *or* `DLHandle` over this ABI.
- **C / C++** — include the header directly (verified).
- **Go** (cgo), **Java/Kotlin** (JNI/uniffi), **C# / .NET** (P/Invoke), **Ruby/PHP** (FFI) — all bind here.

Verified end-to-end from **C** and **Zig** on Qwen2.5-0.5B (Metal): `ferric_generate` → "Paris…",
`ferric_generate_json` → `{"name":"John","age":30}`.
